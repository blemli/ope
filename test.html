Tilde Expansion

TestLinkExpected
Home directory ope://~ open Finder shows home
Downloads ope://~/Downloads open Finder shows ~/Downloads
Desktop ope://~/Desktop open Finder shows ~/Desktop
Documents ope://~/Documents open Finder shows ~/Documents

Blocked Extensions (security policy)

These should be refused with an error dialog. Default blocklist: *.exe, *.bat, *.cmd, *.ps1, *.vbs, *.js, *.msi, *.scr, *.com, *.pif, *.reg, *.wsf, *.wsh
TestLinkExpected
.exe file ope:///tmp/malware.exe blocked Error dialog
.bat file ope:///tmp/script.bat blocked Error dialog
.ps1 file ope:///tmp/payload.ps1 blocked Error dialog
.js file ope:///tmp/script.js blocked Error dialog
.msi installer ope:///tmp/setup.msi blocked Error dialog
.reg file ope:///tmp/hack.reg blocked Error dialog

Unknown Files (ask dialog)

Files not on blocklist or allowlist trigger a 3-button confirmation: Allow Once / Always Allow / Block
TestLinkExpected
.txt file ope:///tmp/notes.txt ask Confirm dialog (if file exists)
.pdf file ope:///tmp/document.pdf ask Confirm dialog (if file exists)
.png image ope:///tmp/photo.png ask Confirm dialog (if file exists)

Non-existent Paths

TestLinkExpected
Missing directory ope:///nonexistent/folder error "Not Found" dialog
Missing file ope:///tmp/doesnotexist.txt error "Not Found" dialog
Missing tilde path ope://~/NoSuchFolder error "Not Found" dialog

Glob / Wildcard Patterns

Wildcards resolve to the first match. Create test files first: touch /tmp/ope-test-1.txt /tmp/ope-test-2.txt
TestLinkExpected
Star wildcard ope:///tmp/ope-test-*.txt ask Opens first match
Question mark ope:///tmp/ope-test-?.txt ask Opens first match
No match ope:///tmp/zzz-no-match-*.xyz error "no files matched" error

URL Encoding

Create test folder: mkdir -p "/tmp/my folder"
TestLinkExpected
Space (%20) ope:///tmp/my%20folder open Opens "my folder"
Localhost form ope://localhost/tmp open Opens /tmp
